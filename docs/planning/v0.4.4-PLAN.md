# v0.4.4 - Database Tools (Comprehensive Plan)

**Target**: Q1-Q2 2026
**Focus**: Complete database lifecycle management from local → staging → production
**Depends on**: v0.4.3 (environments for multi-environment operations)

---

## Executive Summary

v0.4.4 delivers **enterprise-grade database tooling** that matches or exceeds Supabase and Nhost capabilities. The release focuses on:

1. **Migrations** - Schema versioning with rollback support
2. **Backup/Restore** - PITR, S3 storage, cross-environment restore
3. **Mock Data** - Deterministic, shareable, schema-aware generation
4. **Seeding** - Idempotent data seeding with environment awareness
5. **Database Operations** - Shell, queries, cloning, analysis
6. **Schema Management** - Diffing, visualization, index advisor
7. **Type Generation** - TypeScript/Go/Python types from schema
8. **Database Inspection** - Performance analysis tools
9. **Database Testing** - pgTAP integration
10. **Branch Databases** - Isolated DB per git branch

## Command Details

### 1. `nself db` - Database Operations

```bash
# ═══════════════════════════════════════════════════════════════
# MIGRATIONS
# ═══════════════════════════════════════════════════════════════

nself db migrate                    # Run pending migrations
nself db migrate:create <name>      # Create new migration file
nself db migrate:rollback           # Rollback last migration
nself db migrate:rollback --all     # Rollback all migrations
nself db migrate:rollback --steps 3 # Rollback specific number
nself db migrate:status             # Show migration status
nself db migrate:fresh              # Drop all tables, re-run migrations
nself db migrate:refresh            # Rollback all, then migrate

# ═══════════════════════════════════════════════════════════════
# INTERACTIVE SHELL
# ═══════════════════════════════════════════════════════════════

nself db shell                      # Interactive psql session
nself db shell --env staging        # Connect to staging DB
nself db shell --read-only          # Read-only connection

# ═══════════════════════════════════════════════════════════════
# SQL EXECUTION
# ═══════════════════════════════════════════════════════════════

nself db query "<sql>"              # Run SQL query
nself db query -f script.sql        # Run SQL file
nself db query "<sql>" --csv        # Output as CSV
nself db query "<sql>" --json       # Output as JSON

# ═══════════════════════════════════════════════════════════════
# DUMP & LOAD
# ═══════════════════════════════════════════════════════════════

nself db dump                       # Full database dump
nself db dump --schema-only         # Schema only
nself db dump --data-only           # Data only
nself db dump --table users         # Single table
nself db dump --exclude auth.*      # Exclude schema
nself db dump -o backup.sql         # Custom output file

nself db load <file>                # Load dump into database
nself db load <file> --clean        # Drop objects first
nself db load <file> --data-only    # Data only (no schema)

# ═══════════════════════════════════════════════════════════════
# CLONING
# ═══════════════════════════════════════════════════════════════

nself db clone staging              # Clone current DB to staging
nself db clone --from prod          # Clone from production
nself db clone --from prod --anonymize  # Clone with PII anonymization
nself db clone --from prod --to staging # Explicit source/target

# ═══════════════════════════════════════════════════════════════
# ANALYSIS & INSPECTION (from Supabase)
# ═══════════════════════════════════════════════════════════════

nself db status                     # Database health status

nself db size --top 10              # Top 10 largest tables

# Performance analysis
nself db inspect cache-hit          # Check cache efficiency
nself db inspect index-usage        # Analyze index efficiency
nself db inspect unused-indexes     # Find unused indexes
nself db inspect seq-scans          # Count sequential scans
nself db inspect bloat              # Check table/index bloat
nself db inspect locks              # View current locks
nself db inspect blocking           # See blocking queries

# Query analysis
nself db inspect slow-queries       # Show slow query log
nself db inspect outliers           # High execution time queries
nself db inspect long-running       # Currently running slow queries

# Maintenance
nself db optimize                   # Analyze and vacuum
nself db optimize --full            # Full vacuum (locks table)
nself db reindex                    # Rebuild indexes
nself db reindex <table>            # Rebuild specific table indexes

# ═══════════════════════════════════════════════════════════════
# CONNECTIONS
# ═══════════════════════════════════════════════════════════════

nself db connections                # Show active connections
nself db connections --kill <pid>   # Kill specific connection
nself db connections --kill-idle    # Kill idle connections

# ═══════════════════════════════════════════════════════════════
# TESTING (pgTAP integration)
# ═══════════════════════════════════════════════════════════════

nself db test                       # Run all database tests
nself db test <test-file>           # Run specific test
nself db test --schema public       # Test specific schema
nself db test:create <name>         # Create new test file
nself db test:coverage              # Show test coverage

# ═══════════════════════════════════════════════════════════════
# LINTING
# ═══════════════════════════════════════════════════════════════

nself db lint                       # Lint database schema
nself db lint --fix                 # Auto-fix issues
```

## Environment-Aware Behavior

### Automatic Detection

| Environment | Mock Data | Seeding | Real Data | Anonymization |
|-------------|-----------|---------|-----------|---------------|
| **Local** | Allowed | All seeds | Never | N/A |
| **Staging** | Allowed | Test seeds | Via sync only | Required for prod data |
| **Production** | Blocked | Admin seeds only | Yes | N/A |

### Safety Guards

```bash
# These commands are BLOCKED in production:
nself mock generate          # ERROR: Mock data not allowed in production
nself data reset             # ERROR: Data reset not allowed in production
nself db migrate:fresh       # ERROR: Fresh migration not allowed in production

# These require confirmation in production:
nself restore <id>           # Requires --confirm flag
nself db migrate:rollback    # Requires --confirm flag

# Anonymization required for prod → staging:
nself data sync --from prod  # ERROR: Must include --anonymize
```

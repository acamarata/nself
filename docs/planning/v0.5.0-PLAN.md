## Release Process

### Pre-Release (2 weeks before)

1. **Feature Freeze**
   - No new features after this point
   - Only bug fixes allowed

2. **Release Candidate**
   ```bash
   git tag v0.5.0-rc.1
   ```

3. **Community Testing**
   - Announce RC on GitHub, Discord, Twitter
   - Collect feedback for 1 week
   - Address critical issues

4. **Documentation Review**
   - Final review of all documentation
   - Update screenshots/examples
   - Test all code examples

### Release Day

1. **Final Checks**
   ```bash
   # All tests pass
   ./run-tests.sh --all

   # Version updated
   cat src/VERSION  # Should show 0.5.0

   # Changelog updated
   head -50 CHANGELOG.md

   # No uncommitted changes
   git status
   ```

2. **Tag Release**
   ```bash
   git tag -a v0.5.0 -m "v0.5.0 - Full Production Release"
   git push origin v0.5.0
   ```

3. **Build Packages**
   - Homebrew formula
   - npm package
   - Debian package
   - RPM package
   - AUR package
   - Docker image

4. **Publish**
   - GitHub Release with notes
   - npm publish
   - Homebrew tap update
   - Docker Hub push
   - Package repository updates

5. **Announce**
   - GitHub release announcement
   - Twitter/X announcement
   - Discord announcement
   - Blog post
   - Hacker News post (if appropriate)

### Post-Release

1. **Monitor**
   - Watch for critical issues
   - Monitor package downloads
   - Track GitHub stars/forks

2. **Support**
   - Respond to issues quickly
   - Update FAQ as needed
   - Collect feedback for v0.5.1

## Migration Guides

### From Docker Compose

```bash
# If you have existing docker-compose.yml
nself init --from-compose

# Analyzes your compose file
# Creates nself configuration
# Preserves your services
```

### From Kubernetes

```bash
# If you have existing K8s manifests
nself init --from-k8s ./k8s/

# Converts to nself configuration
# Can deploy back to K8s with nself k8s
```

### From Heroku

```bash
# Migration guide for Heroku users
nself init --from-heroku

# Converts Procfile to services
# Maps add-ons to nself services
# Preserves environment variables
```

### From Railway/Render/Fly.io

```bash
# Similar migration paths
nself init --from-railway
nself init --from-render
nself init --from-fly
```
